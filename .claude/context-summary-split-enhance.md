## 项目上下文摘要（分屏模式增强：拖拽调整与同步滚动）
生成时间：2026-02-26 11:53:00

### 1. 相似实现分析
- **实现1**: `src/components/Editor/EditorContainer.tsx:277-293`
  - 模式：使用 Flex 布局并排放置 `SourceEditor` 和 `WysiwygEditor`，中间有 `.editor-split__divider`。
  - 可复用：现有的 CSS 类 `editor-split__*`。
  - 需注意：拖拽改变宽度需要记录鼠标拖拽的 clientX，并调整两侧宽度比例 (flex-basis 或 width)。

### 2. 项目约定
- **命名约定**: 使用小驼峰定义 React 状态和回调函数。
- **文件组织**: 在 `EditorContainer` 中集成视图容器级别的功能。
- **代码风格**: React + TS，Hooks 优先，事件监听注意注销。

### 3. 可复用组件清单
- DOM 事件监听：鼠标拖拽 (`mousemove`, `mouseup`) 和滚动 (`scroll`) 均使用纯原生 DOM 监听绑定在特定 Ref 上。

### 4. 测试策略
- **测试模式**: 编译后本地验证分屏下拖拽中轴线是否改变宽度；在左侧滚动能否带动右侧滚动，点击左侧是否让右侧跳转（比例等高匹配）。

### 5. 依赖和集成点
- **依赖库**: React Hooks

### 6. 技术选型理由
- **为什么用这个方案**: 选用 DOM 比例同步滚动算法最为轻量、对双侧编辑器侵入最小。

### 7. 关键风险点
- **边界条件**: 死循环滚动（需要通过标志位 `isSyncing` 阻断相互触发）；拖拽过小导致某侧编辑器不可逆压缩（需要设定 `min-width` 或拖动限制区域）。
